# Relationship Trigger Tracker

> Layer ID: `MHE.TRACKING.RELATIONSHIP`  
> Purpose: 專門追蹤親密關係、曖昧、被拒絕、忽冷忽熱、回覆慢、被選擇感相關 trigger。

---

## 1. Common Relationship Triggers

| Trigger | Possible Core Theme |
|---|---|
| 對方回覆慢 | 我不重要 / 我被忽略 |
| 對方變冷 | 我會被丟下 |
| 對方有 crush / 喜歡別人 | 我不是被選擇的人 |
| 被拒絕 | 我不夠好 / 我不值得被愛 |
| 對方靠近 | 親密會失控 / 我會受傷 |
| 對方問真心話 | 真實會被拒絕 |
| deep talk 後距離變化 | 被看見後會被丟下 |

---

## 2. Decision Guard

被 relationship trigger 打中時，不即時做：

- 表白
- 斷聯
- 質問
- 長訊息
- 刪對話
- 測試對方
- 故意冷淡

先做：

```txt
1. 情緒強度評分。
2. 寫 draft，不即時送出。
3. 24 小時後再看。
4. 分清楚：我想表達需要，還是想透過訊息解除焦慮？
```

## 4. Message Draft Filter

發訊息前問：

1. 我現在強度幾多？
2. 這訊息是為了表達清楚，還是為了逼對方安撫我？
3. 我能接受對方不即時回嗎？
4. 如果對方不回，我是否會崩潰？
5. 我是否可以簡短、成熟、不控訴地表達？

---

## 5. Safer Message Template

```txt
我這幾日有留意到自己情緒被觸發，所以我想慢慢處理，不想把壓力丟給你。
如果我回覆慢一點，不是嬲你，只是想讓自己穩定一點。
```

---

## 6. Pattern Review

每 14 天問：

- 我最常被哪一種關係訊號打中？
- 我的第一反應是靠近、質問、消失、冷掉，還是討好？
- 我真正需要的是確定性、被選擇、被理解，還是界線？
- 哪些是現在這段關係的真實問題？
- 哪些是舊 pattern 被觸發？
